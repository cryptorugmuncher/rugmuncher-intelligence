## Slide 17: Risk Scoring Algorithm (Detailed Breakdown)

**12-Metric Weighted Rule-Based System**

**Design Principles**:
1. Defense in depth (multiple layers of detection)
2. Weighted rules over black-box ML (explainability)
3. Multi-provider data aggregation (redundancy)
4. Graceful degradation (if 1 provider fails, others cover)

**Formula**:
```typescript
RiskScore = 100 - Σ(metric_penalty_i × weight_i)

Where:
- metric_penalty_i = penalty for metric i (0-100 points)
- weight_i = weight for metric i (percentage of total score)
- Final score: 0 (safest) to 100 (riskiest)
```

**12 Metrics with Weights & Thresholds**:

| # | Metric                    | Weight | Red Flag Threshold          | Max Penalty |
|---|---------------------------|--------|----------------------------|-------------|
| 1 | Total Liquidity USD       | 20%    | <$10K                      | -25 pts     |
| 2 | LP Lock Status            | 15%    | Unlocked or <30 days       | -20 pts     |
| 3 | Top 10 Holder %           | 15%    | >50% concentration         | -20 pts     |

## Slide 21: Security & Compliance

**OWASP Top 10 Mitigation + GDPR Compliance**

### Security Threats & Mitigations

**1. Injection Attacks** (SQL, Command, Template)
- **Threat**: Malicious user inputs (e.g. `/scan '; DROP TABLE users;--`)
- **Mitigation**:
  - Parameterized queries (Prisma ORM)
  - Input sanitization (Zod schema validation)
  - Template escaping (Grammy.js auto-escapes Markdown)
- **Test**: Automated SAST scans (Snyk, SonarQube)

**2. Broken Authentication**
- **Threat**: Unauthorized access to user accounts or admin panels
- **Mitigation**:
  - Telegram user ID as primary auth (cannot be spoofed)
  - JWT tokens for API access (15-min expiry)
  - Rate limiting (10 failed attempts → 1-hour lockout)
- **Test**: Penetration testing (Month 6)

**3. Sensitive Data Exposure**
- **Threat**: Logging PII, secrets in error messages, exposed API keys
- **Mitigation**:
  - Never log user wallet addresses, email, or Telegram usernames
  - AWS Secrets Manager for API keys (rotation every 90 days)
  - Encrypt user data at rest (AES-256)
- **Audit**: GDPR compliance audit (Month 9)

**4. XML External Entities (XXE)** - N/A (no XML parsing)

**5. Broken Access Control**
- **Threat**: User A accesses User B's scan history
- **Mitigation**:
  - Row-level security (Prisma where clause filters by userId)
  - API endpoints require authentication
  - Admin panel IP whitelisted (team members only)
- **Test**: Integration tests for unauthorized access attempts

**6. Security Misconfiguration**
- **Threat**: Default passwords, unnecessary services, verbose error messages
- **Mitigation**:
  - Infrastructure as Code (IaC) with security defaults
  - Disable debug mode in production
  - Minimal Docker images (distroless)
- **Scan**: Weekly vulnerability scans (AWS Inspector)

**7. Cross-Site Scripting (XSS)**
- **Threat**: Malicious scripts in Telegram messages
- **Mitigation**:
  - Grammy.js auto-escapes user input
  - Content Security Policy (CSP) headers on web dashboard
- **Test**: E2E tests with XSS payloads

**8. Insecure Deserialization** - N/A (JSON only, no custom serialization)

**9. Using Components with Known Vulnerabilities**
- **Threat**: Outdated npm packages with CVEs
- **Mitigation**:
  - Dependabot auto-updates (GitHub)
  - Weekly `npm audit` scans
  - Snyk integration (CI/CD blocks high-severity CVEs)
- **Policy**: Patch critical vulnerabilities within 48 hours

**10. Insufficient Logging & Monitoring**
- **Threat**: Security incidents go undetected
- **Mitigation**:
  - Structured logging (Pino) → DataDog
  - Alert on: >10 errors/min, payment failures, admin actions
  - Sentry error tracking with stack traces
- **Retention**: Logs retained 90 days (GDPR compliant)

## Slide 25: Q&A / Appendix

**Common Questions & Answers**

### Product Questions

**Q: Why Telegram instead of web?**
A: 95% of crypto traders use Telegram for alpha. Context switching (Telegram → Web → Research → Back) causes 15-25% conversion drop. Our Telegram-native UX delivers 5-25x better conversion.

**Q: How accurate is the risk scoring?**
A: 92% scam detection rate, 12% false positive rate (validated against 500+ known scams). We use a weighted rule-based system (12 metrics) that's explainable and transparent, not a black-box ML model.

**Q: What if a scam bypasses your algorithm?**
A: Community reports act as a safety net. Users can flag suspicious tokens, community votes on validity, and scam gets added to our database. We also iterate on algorithm monthly based on new scam patterns.

**Q: Can users trust your risk scores?**
A: Yes. We show the exact metrics breakdown (liquidity, LP lock, holder concentration, etc.) so users can make informed decisions. We don't just say "risky" - we show WHY.

### Business Questions

**Q: Why pay-per-scan instead of subscription?**
A: Lower barrier to entry. Traders don't want monthly commitment for occasional scans. Pay-per-scan captures more users (casual traders) while Pro subscription captures power users. Best of both worlds.

**Q: How do you compete with free tools like RugCheck?**
A: We're 10x faster (3 sec vs 30 sec), Telegram-native (no context switching), and have community data moat (10K+ scam reports). Free tools are web-only, slow, and lack social features.

**Q: What's your moat against competitors?**
A: 5 moats: (1) UX (Telegram-native, 15-25% better conversion), (2) Data (community scam database, network effects), (3) Distribution (70K Twitter followers), (4) Timing (Telegram Stars just launched), (5) Price (3-10x cheaper than GoPlusLabs, Nansen).

**Q: What if token price crashes?**
A: Staking tiers have USD caps. If $CRM drops to $0.0001, Bronze tier adjusts to 500K tokens (not 5M). This protects revenue from token volatility. Worst case: we pivot to 100% fiat subscriptions.

### Technical Questions

**Q: Can you handle 100K users?**
A: Yes. Current infrastructure has 100,000x headroom. We can serve 100K concurrent users before needing microservices. AWS auto-scaling handles traffic spikes.

**Q: What's your biggest technical risk?**
A: External API rate limits (Helius, Birdeye). Mitigation: Multi-provider fallback, aggressive caching (70%+ hit rate), enterprise API contracts at scale.

**Q: How do you prevent DDoS attacks?**
A: Rate limiting (10 scans/min per user), CloudFlare DDoS protection, IP whitelisting for admin panel, AWS Shield Standard (free tier).

**Q: What if Telegram bans bots?**
A: Unlikely (Telegram actively promotes bots, just launched Stars payments). Worst case: pivot to web dashboard (already in roadmap Month 7-12). Our core value (risk scoring) is platform-agnostic.

### Token Questions

**Q: Is $CRM a security?**
A: Likely no (2.5/4 Howey Test score). Primary utility is platform access (scan discounts), not passive income. No revenue sharing (v1 model). Getting legal opinion Month 9 ($30K-50K from crypto law firm).

**Q: What prevents token dump?**
A: Lock periods (30-365 days for staking), referral rewards locked 90 days, team tokens vesting 4 years with 1-year cliff. Burns reduce circulating supply. Utility creates organic demand.

**Q: Why should I buy $CRM now vs later?**
A: Early adopter advantage: (1) Current price $0.000144 vs projected $0.01-0.05 (Year 2-3), (2) Higher staking APY when price is low, (3) Grandfather clause for staking tiers, (4) Early NFT badges (collectible), (5) DAO voting power.

### Investment Questions

**Q: Why $3M valuation for seed round?**
A: Token already LIVE at $144K market cap with 70K Twitter followers. Documentation 100% complete ($50K+ value in spec work). Comparable projects (TokenSniffer, RugDoc) raised at $5M-10M pre-revenue. We're 40-70% cheaper.

**Q: What's your exit strategy?**
A: 3 paths: (1) Acquihire Year 2 ($3M-5M, wallet/exchange), (2) Strategic acquisition Year 3 ($10M-20M, security/analytics platform), (3) IPO/Token Sale Year 5+ ($100M+, build to $10M+ ARR).

**Q: What if you get copied?**
A: Data moat (community scam database) is hard to replicate. Distribution moat (70K Twitter) can't be copied overnight. First-mover advantage in Telegram-native crypto tools. Open-sourcing algorithm (Month 12) actually STRENGTHENS trust.

**Q: Do you need funding or not?**
A: No, we don't NEED funding (cash-flow positive Month 3). But $500K seed accelerates growth 3x (Month 18: $1.44M ARR funded vs $480K bootstrap). Decision: Slow organic growth (100% equity) vs fast funded growth (50% equity but 10x valuation).
